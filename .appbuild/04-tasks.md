# 04-Tasks — 365일 말씀 묵상 시집 사이트 (1차 슬라이스)

수직 슬라이스 원칙: 각 작업이 화면+데이터+로직을 함께 관통해 데모 가능한 결과를 낸다.
1차 슬라이스 전체 완료 시 "표지 → 목차 → day_001~003 상세 → prev/next 네비게이션"이
`npm run dev`로 실제 클릭 가능해야 한다(완료 기준은 `05-gate.md`에서 감독관이 파생).

## 작업 목록

### T1. Astro 프로젝트 뼈대 + content collection 스키마
- **대응 기능/화면**: 03-architecture 전체(모든 화면의 토대), F5 전제조건.
- **내용**: `npm create astro@latest`(TS, minimal 템플릿) → `src/content/config.ts`에 `days`
  컬렉션 Zod 스키마 정의(day/title/scriptureRef/scriptureText/scriptureVersion/month/afterword) →
  `src/data/months.ts`에 master_plan.md 12개월 표 코드화 → `src/styles/global.css` CSS 변수
  뼈대 → `BaseLayout.astro` 최소 shell.
- **수용 기준**: `npm run build` 성공(콘텐츠 0개 상태에서도 스키마 정의만으로 빌드 에러 없음).
- **의존**: 없음(최초 작업).

### T2. day_001~003 콘텐츠 마이그레이션
- **대응 기능/화면**: F5.
- **내용**: `../집필부/day_001~003.md`를 03-architecture "마이그레이션 규칙"에 따라
  `src/content/days/day-001.md`~`day-003.md`로 변환(프론트매터 분해 + 시 본문 이관).
- **수용 기준**: 3개 파일 모두 T1의 Zod 스키마 검증을 통과(빌드 에러 없음), 원문과 대조해
  제목·성경구절·시 전문·짧은 여운 텍스트 손실/왜곡 없음(육안 대조).
- **의존**: T1.

### T3. 표지 페이지 (`/`)
- **대응 기능/화면**: F1, 02-screens §2.1.
- **내용**: `src/pages/index.astro` — 제목·부제·목차 링크.
- **수용 기준**: `npm run dev` 후 `/`에서 제목·부제·"목차 보기" 링크가 보이고 클릭 시 `/toc`로 이동.
- **의존**: T1.

### T4. 목차 페이지 (`/toc`) — 활성/비활성 day 목록
- **대응 기능/화면**: F2, 02-screens §2.2.
- **내용**: `src/pages/toc.astro` — `months` 배열 순회 + `days` 컬렉션에서 월별 필터링,
  존재하는 day는 링크, 없는 day(004~365)는 비활성 표시. (비활성 항목은 1~365 전체를
  나열하지 않고, 실용적으로 "각 월의 존재하는 day + 해당 월 나머지는 '(미공개)' 요약
  표기"로 구현 — 365개 개별 비활성 항목을 다 그리는 것은 과설계이자 성능 낭비이므로,
  월별 진행 카운트(예 "1월: 3편 공개")로 축약. 이 구현 판단은 02-screens 원칙("비활성
  시각적 구분")을 충족하면서 실용성을 확보한 것 — 감독관 검증 시 재확인.)
- **수용 기준**: `/toc`에서 12개월 섹션 모두 렌더링, 1월 섹션에 day_001·002·003 활성 링크
  3개가 보이고 클릭 시 각 `/day/1~3`로 이동, 나머지 월은 미공개 상태로 표시.
- **의존**: T1, T2.

### T5. 일일 묵상 페이지 (`/day/[n]`) — 4요소 렌더링
- **대응 기능/화면**: F3, 02-screens §2.3.
- **내용**: `src/pages/day/[day].astro` — `getStaticPaths()`로 `days` 컬렉션 전체 순회,
  제목/성경구절블록/시 본문/여운 렌더링.
- **수용 기준**: `/day/1`, `/day/2`, `/day/3` 각각에서 원문 4요소가 텍스트 손실 없이 보인다.
- **의존**: T1, T2.

### T6. 이전/다음 날 네비게이션 (`DayNav`)
- **대응 기능/화면**: F4, 02-screens §2.3.
- **내용**: `src/components/DayNav.astro`(prevDay?/nextDay? props) 구현 → T5의 `[day].astro`에서
  현재 day±1이 컬렉션에 있는지 조회해 props 전달, "목차로" 링크도 포함.
- **수용 기준**: `/day/2`에 이전(→`/day/1`)·다음(→`/day/3`) 링크 모두 존재, `/day/1`은 다음
  링크만, `/day/3`은 이전 링크만 존재(다음 링크 없음 — 아직 day_004 미작성).
- **의존**: T5.

### T7. 타이포그래피 중심 디자인 다듬기
- **대응 기능/화면**: PRD 비기능 요구(시가 주인공, 미니멀·여백), 모든 화면.
- **내용**: `global.css`에 서체 스케일·행간·본문 max-width·반응형(모바일) 미디어쿼리 적용.
  장식 요소(그림자·그라디언트·아이콘) 최소화.
- **수용 기준**: 데스크톱·모바일 뷰포트(브라우저 리사이즈로 확인) 양쪽에서 시 본문 줄바꿈이
  깨지지 않고 여백이 유지된다.
- **의존**: T3, T4, T5, T6(모든 페이지가 존재해야 전체 다듬기 가능).

## 순서·의존 그래프
```
T1 → T2 → T4 ┐
      └→ T5 ─┴→ T6 → T7
T1 → T3 ──────────────┘
```
(T3는 T2와 무관 — 독립적으로 T1 직후 착수 가능)

## 검증 방법 (E2E 대체 — 프로젝트 규모에 맞는 축소)
1인·3페이지 슬라이스 규모라 자동화 E2E 테스트 프레임워크(Playwright 등)는 과설계로 판단,
아래 **수동 브라우저 검증 절차**를 이번 슬라이스의 공식 E2E 대체 기준으로 삼는다
(05-gate.md 증거 칸은 이 절차의 실행 결과로 채운다):
1. `npm run build` — 에러 0, 산출물이 순수 정적 HTML(서버 함수 없음) 확인.
2. `npm run dev`(`cys run --scoped`) 기동 후 브라우저에서 `/` → `/toc` → `/day/1` →
   (다음 날)`/day/2` → (다음 날)`/day/3` → (이전 날 2회로 `/day/1` 복귀) → `/toc`까지
   전체 동선을 실제로 클릭해 이동 확인.
3. `/day/1`~`/day/3` 각각에서 원문(`../집필부/day_00N.md`)과 화면 텍스트를 육안 대조
   (제목·성경구절·시 전문·짧은 여운 손실/왜곡 없음).

## 명명 규칙 참고
원본 콘텐츠 파일은 `day_00N.md`(언더스코어, 집필부 컨벤션), 사이트 content collection은
`day-00N.md`(하이픈, Astro 파일라우팅 관례)로 **의도적으로 다르다** — 혼동 방지를 위해 명시.
