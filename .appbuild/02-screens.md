# 02-Screens — 365일 말씀 묵상 시집 사이트

## 1. 화면 흐름 (IA)

```
/ (표지, F1)
  └─ "목차 보기" 클릭 → /toc

/toc (목차, F2)
  ├─ 활성 day 링크 클릭 → /day/[n]
  └─ (표지로 돌아가기 링크) → /

/day/[n] (일일 묵상, F3+F4)
  ├─ "이전 날" → /day/[n-1]  (n=1이면 링크 없음)
  ├─ "다음 날" → /day/[n+1]  (n=최신 공개 day면 링크 없음)
  └─ "목차로" → /toc
```

기능↔화면 매핑(PRD 01 대조):
| PRD 기능 | 화면 |
|---|---|
| F1 표지 | `/` |
| F2 목차(12개월) | `/toc` |
| F3 일일 페이지 4요소 | `/day/[n]` |
| F4 이전/다음 네비게이션 | `/day/[n]` 하단 |
| F5 콘텐츠 마이그레이션 | (화면 아님 — 데이터 파이프라인, 03-architecture 참조) |

누락 없음: PRD의 F1~F4가 모두 화면에 대응됨.

## 2. 화면별 상세

### 2.1 `/` — 표지

**레이아웃**: 뷰포트 중앙 정렬, 세로 여백 넉넉하게(시집 표지 느낌 — 텍스트만으로 구성,
이미지 없음).
- 시집 제목 "365일 말씀 묵상 시집" (가장 큰 서체, 세리프 계열 권장)
- 부제(1~2문장, PRD 목적 요약): "성경 구절 하나를 매일 한 편의 시로 만나는 묵상 시집"
- "목차 보기 →" 링크(표지의 유일한 CTA)

**상태**:
- 정상: 위 3요소 렌더링.
- 빈/에러/로딩: 해당 없음(정적 콘텐츠, 데이터 의존 없음 — 항상 동일하게 렌더링).

**컴포넌트**: `BaseLayout`(공통 shell) 안에 인라인 마크업. 별도 재사용 컴포넌트 불필요
(1차 슬라이스 기준 단순 페이지).

**데이터**: 없음(하드코딩된 정적 텍스트).

---

### 2.2 `/toc` — 목차

**레이아웃**: 세로로 12개월 섹션이 순서대로 나열. 각 월 섹션:
- 월 헤더: "1월 — 시작: 태초의 빛" (month + theme, `months.ts`)
- 하위에 해당 월 소속 day 리스트. 각 항목은 "Day 001. 태초에, 라는 말" 형식(day 번호 + 제목).
  - **활성**(콘텐츠 존재): 클릭 가능한 링크, `/day/[n]`로 이동.
  - **비활성**(콘텐츠 미존재, 004~365): 흐린 색(회색 텍스트), 클릭 불가(링크 아님, `<span>`),
    "미공개" 또는 시각적으로 구분되는 스타일만으로 충분(별도 배지 텍스트는 선택사항).

**상태**:
- 정상: `days` 컬렉션에 1개 이상 있을 때 — 위 레이아웃대로 활성/비활성 혼합 렌더링.
  1차 슬라이스 실측: day_001~003 활성, 004~365 비활성.
- 빈(컬렉션 자체가 0개일 때): 12개월 섹션 헤더는 그대로 보이되 모든 day가 비활성 —
  별도 "아직 콘텐츠가 없습니다" 안내 문구는 이번 슬라이스에서 필수 아님(항상 최소 1개
  day는 존재하는 게 실제 운영 전제이므로 과설계 안 함). 단, 렌더링 자체가 깨지지 않아야
  한다(빈 배열에서도 월 섹션 12개는 정상 출력).
- 에러/로딩: 해당 없음(빌드타임 정적 생성, 런타임 fetch 없음).

**컴포넌트**: 목차 페이지 전용 마크업(월 섹션 반복 렌더링). 재사용 필요성이 낮아 별도
컴포넌트 분리는 과설계 — `toc.astro` 내부에서 `months` 배열을 순회하며 인라인 처리.

**데이터**: `src/data/months.ts`(12개월 메타) + `days` 컬렉션 전체(month별 필터링).

---

### 2.3 `/day/[n]` — 일일 묵상 페이지

**레이아웃**: 세로 1단, 좁은 본문 너비(가독성 위해 max-width 제한, 예: 640px 내외)로 중앙 정렬.
순서:
1. 제목 (예: "Day 001. 태초에, 라는 말")
2. 성경 구절 블록(출처 라벨 + 인용문 + 영문/버전 표기) — blockquote 스타일로 시각적 구분
3. 묵상시 본문(연 구분 유지, 넉넉한 줄간격·행간)
4. 짧은 여운(afterword) — 시 본문과는 다른 톤(더 작은 서체 크기 또는 이탤릭)으로 시각적 구분
5. `DayNav` 컴포넌트(이전/다음 날 + 목차로 가기)

**상태**:
- 정상: 위 5블록 모두 렌더링, day 번호가 `days` 컬렉션에 존재.
- 빈: 해당 없음(day 자체가 없으면 이 라우트가 애초에 `getStaticPaths`에서 생성되지 않음).
- 에러(존재하지 않는 day 번호로 접근, 예: `/day/999`): Astro 정적 라우팅 특성상
  `getStaticPaths`가 생성한 경로 외에는 자동으로 404 — 별도 커스텀 404 페이지는 이번
  슬라이스 범위 밖(Astro 기본 404 동작으로 충분, PRD에 명시된 요구 아님).
- 로딩: 해당 없음(정적 HTML).

**컴포넌트**:
- `DayNav.astro` — props: `prevDay?: number`, `nextDay?: number`. 둘 다 optional —
  없으면 해당 방향 링크 미출력(F4 경계 처리).

**데이터**: `days` 컬렉션의 단일 엔트리(현재 day) + prev/next 존재 여부 판정(day-1, day+1이
컬렉션에 있는지 조회).

## 3. 공통 (BaseLayout)
모든 화면이 공유: `<html lang="ko">`, 전역 CSS(`global.css` — CSS 변수: 배경/전경색,
서체 스택, spacing 스케일), 상단에 사이트명으로 돌아가는 최소 헤더(표지 링크). 장식 요소
최소화(로고 이미지·아이콘 없음 — 타이포그래피 중심 원칙, PRD 비기능 요구 대응).
